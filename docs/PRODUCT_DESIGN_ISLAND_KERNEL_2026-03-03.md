# OmniAgent 灵动岛产品设计文档（内核版）

日期：2026-03-03  
状态：对齐版（基于代码现状 + 历史 session 共识）  
适用范围：OmniAgent 桌面灵动岛体验 + OmniAgent 工具执行产品主线

---

## 1. 一句话定义

OmniAgent 不是“会聊天的工具箱”，而是“以灵动岛为入口的结果交付系统”：用户一句话或一个拖拽，系统直接交付可用结果。

---

## 2. 已确认的核心共识（来自 2026-02-28 至 2026-03-03 会话）

1. 口号与方法论已经明确：`多快好省`。  
2. 产品风格明确：`小而美、轻量但强大、少即是多`。  
3. 交互主张明确：`点击即聊天`，不是先进入复杂工具页。  
4. 工作方式明确：`拖拽即处理`，例如 PDF 拖入后直接下达“转 Word”并尽快出结果。  
5. 价值取向明确：`不废话，只交付`，能用 100 token 解决的问题绝不走 1000 token 路径。  
6. 形态判断明确：灵动岛是“入口与状态窗口”，不是把全部复杂交互堆在前台。

---

## 3. 产品本质

OmniAgent 要解决的不是“AI 能不能回答”，而是“用户任务能不能马上完成”。

本质上是两层产品：

1. B 端能力层：给其他 Agent 和开发者提供统一工具执行能力（连接 + 执行 + 路由）。
2. C 端体验层：给最终用户一个几乎零学习成本的入口（灵动岛），让复杂能力在后台发生。

灵动岛在这里承担的是“产品门面 + 状态中枢”，而不是另一个复杂应用。

---

## 4. 北极星体验（North Star）

### 北极星路径

用户看到灵动岛 -> 点击进入常驻聊天入口 -> 拖入文件/输入一句任务 -> 系统执行 -> 返回可下载或可确认结果 -> 用户继续下一步

### 北极星标准

1. 用户不需要理解工具名、参数和流程。  
2. 用户总能看到当前状态：待处理、处理中、已完成、失败。  
3. 任务完成后必须有“可行动结果”，而不是一段解释。  
4. 整体流程应像“系统能力”而不是“聊天表演”。

---

## 5. 产品设计原则（内核）

1. 入口极简：入口只做一件事，收任务。  
2. 能力隐藏：复杂度留在后台，前台只暴露必要状态与结果。  
3. 结果优先：优先交付文件、链接、执行确认，不优先长文本。  
4. 状态诚实：处理中就明确处理中，失败就明确失败并给下一步。  
5. 动效有意义：动画用于表达状态变化，不是装饰。  
6. 连续性：一次任务结束后，用户无需重新“找入口”。

---

## 6. 当前进展（按产品层面）

1. 已形成 Web + Desktop 双轨：Web 侧有较完整工具与组件体系，Desktop 侧已建立灵动岛与主面板联动基础。  
2. 灵动岛方向已从“功能堆叠”转向“形态与交互重构”，包括三态思想、场景化适配和交互链路修正。  
3. 历史痛点已被识别并进入方案化处理：例如“展开后无法点击收起”“形态反直觉”等问题，已形成明确整改方向。  
4. 团队共识已从“做更多页面”转向“做更强入口”：入口轻、结果快、体验精。

---

## 7. 下一步要做的事情（产品优先级）

### P0：打穿单一闭环（必须先完成）

1. 固化唯一主入口：点击灵动岛默认进入聊天处理入口，不分散到工具导航。  
2. 固化唯一主流程：拖入文件 + 一句话指令 + 结果交付，确保连续稳定。  
3. 固化状态语言：等待、处理中、完成、失败四态在视觉和文案上完全统一。  
4. 固化收起与回归机制：任务完成后用户始终知道“下一步在哪”。

### P1：做出“苹果感”但不做“苹果壳”

1. 动效编排围绕任务节奏，不围绕炫技。  
2. 每类高频任务有独立微交互反馈（文件、音乐、通知、工具执行）。  
3. 压缩视觉噪音，保留可感知的品质细节（节奏、留白、层次、过渡）。

### P2：建立规模化价值

1. 把“灵动岛入口”与“Universal Tool API”打通为一个叙事。  
2. 把连接能力（OAuth）和工具执行能力统一成“一个入口，一套结果”。  
3. 围绕真实高频场景扩展能力包：文档处理、媒体处理、常用自动化。

---

## 8. 未来 6 周建议节奏（非技术版）

1. 第 1-2 周：只盯一个目标，打穿“点击/拖拽 -> 结果”的最短路径。  
2. 第 3-4 周：统一视觉与动效语言，完成关键场景体验一致性。  
3. 第 5-6 周：扩大到更多真实任务，验证复用性和稳定性，并开始外部可演示版本。

---

## 9. 成功标准（产品指标）

1. 首次任务完成时间显著缩短。  
2. 任务完成率持续提升。  
3. 用户在一次会话内可连续完成多个任务，而不迷失入口。  
4. “解释型回复”占比下降，“可行动结果”占比上升。  
5. 用户主观反馈聚焦在“快、顺、少负担”。

---

## 10. 这版文档的结论

OmniAgent 当前最重要的不是“再加功能”，而是把“灵动岛入口化 + 结果交付化”做到极致。  
只要这个内核成立，工具扩展、连接扩展、商业扩展都会自然发生；反之，功能越多，体验越散。
